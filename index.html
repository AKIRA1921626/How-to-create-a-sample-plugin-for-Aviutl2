AviUtl2 入力プラグイン サンプルプログラム実行手順

今回の記事では、AviUtl2の「入力プラグインのサンプルプログラム」を実行する手順を解説します。ここでは「サンプルプログラムを実行する」ことだけを目的とするため、プログラミングの知識などは一切必要としません。今回は「C++」を使用しますが、C++の書き方自体はサンプルプログラムの実行が済んだ後から勉強をすればよいため、ここではそういうものとして扱います 。

執筆者:AKIRA

1. 必要なもの

今回の手順で必要なものは以下の3つです 。

  • Visual Studio 2026 (VS2026)
    https://visualstudio.microsoft.com/ja/
  • aviutl2_sdk.zip
    https://spring-fragrance.mints.ne.jp/aviutl/
  • AviUtl2本体（動作確認用）
    https://spring-fragrance.mints.ne.jp/aviutl/
開発環境に関する補足：
本手順は、先行事例である「【aviutl2】Plugin開発備忘録｜SDK構成とDLLビルド手順」(2025年7月24日投稿) の流れをベースにしています 。先行記事ではVisual Studio 2022が使用されていますが、本記事では最新の安定版であるVisual Studio 2026を使用します 。不安な場合や、参考記事と完全に同一の手順を行いたい場合はVisual Studio 2022を利用してください 。

2. プラグイン開発における言語と環境の前提

プラグインは、C++などのプログラミング言語で作成したコードを「コンパイル（パソコンが実行できるファイル形式に変換すること）」することで利用可能になります 。C#、C言語、Rust、Delphiなどでも作成可能ですが、公式サンプルのプラグインがC++で記述されているため、今回はC++を採用します （Lua単体でのプラグイン出力はできません ）。C++によるコンパイルを最も簡単に行う手段として、Visual Studioを利用するのが合理的です 。

3. 開発環境のインストール手順

  1. VisualStudioSetup.exe を起動します 。(これはVisual Studio 2026をダウンロードした時に入手可能なインストールファイルです。)
  2. 「続行」をクリックします 。
  3. インストールの詳細画面が表示されたら、「デスクトップとモバイル」項目内にある「C++によるデスクトップ開発」を選択します。このとき、インストールの詳細は変更を行わずにデフォルトのままとします 。
  4. 「インストール」をクリックし、完了まで待ちます（20分程度かかります） 。
  5. インストール完了後、Visual Studioを起動します 。
  6. Microsoftアカウント、またはGitHubアカウントでのログインが要求されるため、ログインします（後から変更やスキップも可能です） 。
  7. 開発設定（変更なし）と配色テーマ（任意）を選択して開始します 。

4. プロジェクトの新規作成と初期設定

  1. 起動画面から「新しいプロジェクトの作成」を選択します 。
  2. テンプレート一覧から「ダイナミック リンク ライブラリ (DLL)」を選択し、「次へ」をクリックします 。
  3. プロジェクト名を任意のものに変更し（今回は Sample とします）、それ以外は変更せず「作成」をクリックします。画面に表示される dllmain.cpp が、後の Sample.dll の中身となります 。
  4. プロジェクトが開かれたら、画面上部のツールバーにある「Debug」の隣を確認し、x86 であればx64に変更します（最初から x64 の場合はそのままにします）。AviUtl2は64bit用に書かれたプラグインしか動作しないため、この設定は必須です 。
  5. 次に、不要なエラーを防ぐ設定を行います。プロジェクトを右クリックして「プロパティ」を開き、「構成プロパティ」→「C/C++」→「プリコンパイル済みヘッダー」 を選択します。一番上の「プリコンパイル済みヘッダー」の項目を**「プリコンパイル済みヘッダーを使用しない」**に変更し、OKボタンを押します。サンプルコードをそのまま貼り付けて動かす際に、システム特有のエラーが出るのを防ぐために必要な設定です。

5. SDKヘッダーファイルの組み込み

  1. ダウンロードした aviutl2_sdk.zip を解凍しておきます 。
  2. VS2026に戻り、画面右側の「ソリューション エクスプローラー」から作成したプロジェクト（Sample）を右クリックし、「エクスプローラーで開く」を選択します 。
  3. 開いたプロジェクトのディレクトリ内に、解凍したSDK内にある拡張子 .h のファイルをコピーします 。今回は入力プラグインを対象とするため、input2.h をここにコピーします 。
  4. VS2026のソリューションエクスプローラー上で、プロジェクトを右クリック ＞「追加」 ＞「既存の項目」を選択し、先ほどコピーした input2.h をプロジェクトに追加します 。
    （※プロジェクトの構成によっては追加せずとも動作する場合がありますが、将来的なファイル構成変更時のエラーを防ぐため、追加しておくことを推奨します 。）

6. サンプルコードの実装とビルド

  1. SDK内のサンプルソースである AviReader.cpp を開き、その全内容をコピーします 。
  2. VS2026上の dllmain.cpp の中身を、コピーした内容で完全に上書きします 。このコードは、先ほど追加した input2.h とシステム用の windows.h を読み込む、入力プラグインのサンプルロジックです 。
  3. ビルドを行うため、ツールバーの「Debug」「x64」となっている箇所のドロップダウンを、「Release」「x64」に変更します 。
  4. メニューから「ビルド」 ＞ 「ソリューションのビルド」を実行します 。
  5. ビルド完了後、プロジェクト名を右クリックして「エクスプローラーで開く」を選択し、表示されたディレクトリの1つ上の階層にある x64 フォルダを開きます 。
  6. その中の Release フォルダ内に、プロジェクト名と同名の Sample.dll が生成されていることを確認します 。

7. 配置と動作確認

  1. 生成された Sample.dll の拡張子を手動で変更し、入力プラグインの形式である Sample.aui2 に書き換えます 。
  2. 変更したファイルを、AviUtl2のプラグイン認識ディレクトリ（例：C:\ProgramData\aviutl2\Plugin、または本体フォルダ内の plugins ディレクトリ）にコピーします 。
  3. AviUtl2を起動し、メニューの「その他」 ＞ 「プラグイン情報」を確認します 。
  4. 一覧に「AVI File Reader(sample)」が表示されていれば、サンプルプラグインのビルドと組み込みは成功です 。

8. プラグイン情報の変更と再ビルド

  1. プラグイン名や識別情報を固有のものに変更するため、VS2026で dllmain.cpp を開きます 。
  2. 「入力プラグイン構造体定義」が記述されているセクションを探し、文字列の値を任意の内容に書き換えます 。
    3. 注意点：
    文字列の先頭にある L および ""（ダブルクォーテーション）は削除しないでください 。L は型を「ワイド文字列」というAviUtl2のAPI仕様に適応させるための識別子であり、削除すると型不一致によるコンパイルエラーが発生します。
  3. 変更後、再度手順6〜7と同様の手順で「Release / x64」ビルドを行い、拡張子を .aui2 に変更して上書き配置します 。
  4. AviUtl2を再起動し、プラグイン情報に変更した名称が反映されていることを確認します 。

これにてサンプルプラグインの作成および変更の反映手順は完了です 。

内部の処理ロジックの書き換えや独自のフィルタ機能を実装する場合、ここから本格的な「C++」の学習へと進むことになります 。
